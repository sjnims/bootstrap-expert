Installation

Install the package using your preferred package manager:

npm install @example/component-library

Or using yarn:

yarn add @example/component-library

After installation, import the library in your project's entry point.

Usage

Basic usage is straightforward. Import the components you need:

import { Button, Card, Modal } from '@example/component-library';

Then use them in your application:

<Button variant="primary">Click me</Button>

All components accept a variety of props for customization.

API Reference

Each component exposes a consistent API. Common props include:

  • className - Additional CSS classes
  • style - Inline styles object
  • variant - Visual variant (primary, secondary, etc.)
  • size - Size variant (sm, md, lg)

See individual component documentation for specific props.

JavaScript Initialization

While scrollspy works via data attributes, you can also initialize or refresh it via JavaScript.

Initialize via JavaScript
const scrollSpy = new bootstrap.ScrollSpy(document.body, {
  target: '#navbar-example',
  rootMargin: '0px 0px -40%',
  smoothScroll: true
});
Refresh after DOM changes
// When adding/removing elements dynamically
scrollSpy.refresh();
Using data attributes
<div data-bs-spy="scroll"
     data-bs-target="#navbar-example"
     data-bs-root-margin="0px 0px -40%"
     data-bs-smooth-scroll="true"
     tabindex="0">
  ...
</div>

Configuration Options

Key data attributes for configuring scrollspy behavior.

Attribute Description Example
data-bs-spy="scroll" Enables scrollspy on the element Required on scrollable container
data-bs-target CSS selector of navigation to update #navbar-example
data-bs-root-margin Intersection Observer rootMargin for activation threshold 0px 0px -40%
data-bs-smooth-scroll Enable smooth scrolling when clicking nav links true

Accessibility Notes

Accessibility Requirements
  • Add tabindex="0" to the scrollable container for keyboard accessibility
  • Ensure section headings have unique id attributes matching navigation href values
  • Navigation links should use proper anchor syntax (href="#section-id")
  • Consider users who navigate with keyboard - they should be able to tab to and scroll the container
  • The data-bs-smooth-scroll option respects prefers-reduced-motion automatically
